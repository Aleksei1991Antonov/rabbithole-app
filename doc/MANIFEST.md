# 📜 МАНИФЕСТ ПРОЕКТА: Кроличья Нора (Rabbit Hole)

> "Everything is a deep dive. Истина где-то рядом."

## 🎯 1. Миссия и Цель
Интеллектуальное путешествие для расширения сознания, в процессе которого негативные флешбеки прошлого обесцениваются и перестают мучить человека.
- **Ключевые ценности:** Скорость, минимализм, наглядность.
- **Платформа:** MiniApp в мессенджере MAX.

## 🛠 2. Технологический Стек
- **Среда:** WebStorm + Vite
- **Frontend:** React + TypeScript
- **Backend/DB:** VK Cloud
- **Styling:** Tailwind CSS + Lucide Icons
- **AI:** GigaChat
- **Интеграции:**
    - API MAX (Авторизация, Соглашения)
    - API GigaChat (Ядро ИИ)

*Обновлено: 13.04.2026 | Статус: Разработка UI завершена, переход к логике.*
*Предпалагаемая дата релиза: 23.04.2026*

Этап 1. Подготовка инфраструктуры и доступов

Зарегистрировать на новый номер MAX.
Почему сначала: Это основа для всех дальнейших действий, связанных с MAX.

Добавить MAX аккаунт в техподдержку.
Почему: Чтобы оперативно решать проблемы, если они возникнут при интеграции.

Загрегистрировать Platform V.
Почему: Это твоя основная платформа для разработки и развертывания.

Создать БД (Базу данных).
Почему: Без базы данных не получится хранить пользователей, их прогресс, рефералов и награды.

Доделать Базу знаний.
Почему: Это поможет тебе и команде быстрее ориентироваться в проекте, а также пригодится для поддержки пользователей.

Этап 2. Разработка и интеграция (Ядро продукта)

Подключить API СБЕР ГИГА ЧАТ.
Почему: Это основной функционал, который ты интегрируешь.

Отладить промпт.
Почему: Качество ответов нейросети напрямую зависит от промпта. Лучше сделать это до запуска бота.

Подключить Sber API.
Почему: Если это дополнительные сервисы (например, для авторизации или платежей), их нужно подключить на этом этапе.

Этап 3. Сборка и запуск (Frontend/UI)

Развернуть miniapp в Platform V.
Почему: Создаешь «скелет» приложения на платформе.

Создать Бот мини-апп в MAX и выложить MiniApp.
Почему: Теперь у тебя есть место, куда будут приходить пользователи.

Подключить авторизацию с MAX.
Почему: Безопасность и идентификация пользователя — критически важный шаг перед запуском основных механик.

Этап 4. Бизнес-логика и монетизация (Финальные штрихи)
Подключить удаление аккаунта.

Почему: Это требование законодательства о персональных данных (GDPR/ФЗ-152). Лучше сделать это до публичного запуска.

Реализовать Реферальную систему + награды.
Почему: Механика привлечения пользователей, запускается после того, как базовый функционал стабилен.

Награда Архитектор.
Почему: Это, скорее всего, игровая механика или ачивка, которая выдается за определенные действия (возможно, за рефералов). Внедряется в самом конце.
